USAGE OPTIONS
═══════════════════════════════════════════════════════════════════════════════

Complete test (default):
  ./test_svn2git.sh
  → Creates SVN, migrates, validates, cleans up
  → Duration: 12-17 minutes

Keep test artifacts for inspection:
  ./test_svn2git.sh --keep-svn --keep-git
  → Preserves both SVN and Git repos
  → Useful for iterative testing

Reuse existing SVN (for testing migration changes):
  ./test_svn2git.sh --skip-svn --verbose
  → Skips SVN creation, runs migration, validates
  → Saves 3-5 minutes per run

Run validation only:
  ./test_svn2git.sh --skip-svn --skip-migrate
  → Tests validation logic independently
  → Useful for testing custom rules

Clean mode (for CI/automated testing):
  ./test_svn2git.sh --clean
  → Auto-removes previous runs without prompting
  → Non-interactive, suitable for CI/CD

Verbose logging (for debugging):
  ./test_svn2git.sh --verbose
  → Shows detailed progress for each stage
  → Useful when troubleshooting
